# Stancard — Real Historical Sparkline Data

## Current State

The Markets tab shows sparkline charts on every stock, forex, and crypto card. All sparklines use mock data generated by `generateSparklineData()` in `Sparkline.tsx` — a seeded PRNG that derives a fake 7-point trend from the current price and change percentage. This means sparklines never reflect real historical price movement.

The backend (`main.mo`) has no historical price endpoint or cache. There are no `HistoricalPrice` types, no `historicalCache` HashMap, and no FMP historical endpoint being called.

## Requested Changes (Diff)

### Add
- `HistoricalPrice` type in `main.mo`: `{ date: Text; close: Float }`
- `HistoricalCache` type in `main.mo`: `{ prices: [HistoricalPrice]; fetchedAt: Int }`
- `historicalCache` stable HashMap in `main.mo`: maps `symbol: Text` → `HistoricalCache`
- `getHistoricalPrices(symbol: Text) : async [Float]` public function in `main.mo`:
  - Checks if cached data exists and is less than 24 hours old (fetchedAt + 86_400_000_000_000 > Time.now())
  - If fresh, returns cached close prices as `[Float]`
  - If stale/missing, calls FMP endpoint: `https://financialmodelingprep.com/api/v3/historical-price-full/{symbol}?timeseries=7&apikey=sNm7MnB0SR8L0Cm2nsK4FwCRFoouf6iZ`
  - Parses JSON response to extract last 7 `close` values in chronological order
  - Stores result in `historicalCache` with current timestamp
  - On any parse/HTTP failure, returns empty array `[]`
- New entry in `backend.d.ts`: `getHistoricalPrices: (symbol: string) => Promise<number[]>`
- Historical data fetching in `MarketsScreen.tsx`: after market data loads, fetch historical prices for all 10 stocks + 3 cryptos (13 symbols) in parallel, store in a `historicalData` state map

### Modify
- `StockCard`, `CryptoCard`, `makeStockItem`, `makeCryptoItem` in `MarketsScreen.tsx`: use real historical close prices from `historicalData[symbol]` if available and non-empty; fall back to `generateSparklineData()` if not
- `ForexCard`, `makeForexItem` in `MarketsScreen.tsx`: keep using `generateSparklineData()` unchanged
- Historical fetching only triggered once when market data first loads, not on every 60-second refresh — re-fetch only if cached data is older than 24h (canister handles this guard)

### Remove
- Nothing removed — mock data generation is kept as fallback

## Implementation Plan

1. **Backend (`main.mo`):**
   - Add `HistoricalPrice` and `HistoricalCache` types
   - Add stable `historicalCache` HashMap
   - Add `parseHistoricalPrices(json: Text) : [Float]` private helper — extracts `close` fields from FMP's `historical` array, returns 7 values oldest-first
   - Add `getHistoricalPrices(symbol: Text) : async [Float]` public function with 24h cache logic, HTTP outcall, parse, store, and fallback-to-empty-array on failure

2. **Backend bindings (`backend.d.ts`):**
   - Add `getHistoricalPrices: (symbol: string) => Promise<number[]>` to the interface

3. **Frontend (`MarketsScreen.tsx`):**
   - Add `historicalData` state: `Record<string, number[]>`
   - Add `historicalLoaded` ref to prevent re-fetching on every 60s refresh
   - After market data is fetched, fetch historical prices for the 13 stock+crypto symbols in parallel using `Promise.all`
   - Pass real data to `StockCard`, `CryptoCard`, `makeStockItem`, `makeCryptoItem` — use it when non-empty, fall back to mock when empty
   - Forex cards untouched
